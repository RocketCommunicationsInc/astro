# Clock

Clock shows the current or provided date and time, and optional AOS and LOS timers. It will typically be positioned on the Global Status Bar.

## Guidelines

* [Astro UXDS: Clock](https://www.astrouxds.com/ui-components/clock)

export const Default = (args) => {
    return html`
<div style="padding: 10%; display: flex; justify-content: center;">
    <rux-clock
        .aos="${args.aos}"
        .los="${args.los}"
        .timezone="${args.timezone}"
        ?hide-timezone="${args.hideTimezone}"
        ?hide-date="${args.hideDate}"
        ?hide-labels="${args.hideLabels}"
        ?small="${args.small}"
        date-in="${args.dateIn}"
        ?static="${args.static}"
    ></rux-clock>
</div>
    `
}

## Variants

### With AOS/LOS Timestamps

> NOTE: The AOS and LOS timestamps accept a number of different time formats. If you don't provide a timezone ("1988-04-22 12:12:12"), they will default to the user's system time zone.
> It is recommended to use the following format `"1988-04-22T12:12:12.000Z"`.

### Static

If you need full control over the time, you can use the `static` attribute and pass in your own time via the
`date-in` attribute. `static` will prevent the clock from ticking.

export const Static = (args) => {
    return html`
<div style="padding: 10%; display: flex; justify-content: center;">
    <rux-clock
        .timezone="${args.timezone}"
        .aos="${args.aos}"
        .los="${args.los}"
        ?hide-timezone="${args.hideTimezone}"
        ?hide-date="${args.hideDate}"
        ?hide-labels="${args.hideLabels}"
        ?small="${args.small}"
        ?static="${args.static}"
    ></rux-clock>
</div>
    `
}
